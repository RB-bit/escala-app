tests spec

Cada proyecto registra tests antes de lanzar cualquier cambio significativo. Un test es una acción + una hipótesis: se evalúa 3–7 días después del lanzamiento para determinar si funcionó o no. El objetivo es construir una base de datos histórica de aprendizajes por marca. El tab "Tests" vive dentro del dashboard de cada proyecto.

flujo del formulario (4 pasos)
1
Tipo de test
Selector visual: meta_ads · google_ads · product_launch · web_change. Determina los campos del paso 3.
2
Info básica (todos los tipos)
Título · Hipótesis · Métricas de éxito · Fecha de lanzamiento (default: hoy) · Fecha de evaluación (default: +7 días, rango: +3 a +14).
3
Campos específicos por tipo
meta_ads: lote (auto), concepto, persona, deseo, ángulo, positioning (dropdown 9 opciones), link ad set, formatos (checkbox múltiple). google_ads: ídem + tipo campaña + link. product_launch: nombre, URL, precio, canales. web_change: página, URL, tipo de cambio, descripción.
4
Confirmación
Resumen de todos los campos. "Lanzar test" → status active. "Guardar borrador" → status draft (no cuenta en métricas).
flujo de evaluación
1
Trigger automático (cron diario)
Cuando evaluate_at llega, el test pasa de active → evaluating. El AM ve un badge con el número de tests pendientes en el tab Tests.
2
Formulario de evaluación
¿Funcionó? (radio: winner · loser · neutral · no_data) — Aprendizajes (textarea, requerido) — Link al anuncio ganador/perdedor (solo meta_ads / google_ads).
3
Cierre
Al guardar: status → closed. Si result = winner y tipo es ads: crear registro en winning_ads + mostrar modal para completar hook y análisis.
Tests cerrados: el AM puede editar learnings y result pero no cambiar el status. No se puede cerrar sin result + learnings.
vistas del tab "Tests"
vistacontenido
ActivosTests active + evaluating, ordenados por evaluate_at asc. Los evaluating aparecen primero con badge de alerta.
HistorialTests closed. Filtros por tipo, resultado y rango de fechas.
Ads GanadoresTabla de winning_ads: lote · hook · ángulo · link · por qué funcionó.
Hit Ratewinners / (winners + losers). Mostrar total de tests cerrados desglosado por tipo. Draft excluidos.
reglas de negocio
  • No se puede cerrar un test sin result y learnings.
  • Tests en estado draft no cuentan en ninguna métrica.
  • Hit rate: winners / (winners + losers). neutral y no_data excluidos del denominador.
  • lot_number es por marca (contador independiente por brand_id). Se guarda como text para admitir sufijos (ej: "146b").
  • Solo editor u owner pueden crear/evaluar tests. viewer es solo lectura.
  • Un test no puede volver a active una vez cerrado.
  • winning_ads solo se genera desde meta_ads y google_ads.
